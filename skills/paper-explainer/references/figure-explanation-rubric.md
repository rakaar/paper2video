# Figure Explanation Rubric

A figure explanation must teach the viewer how the figure makes the paper's point.

## Required Fields

For each figure, capture:

- `figure_id`: figure label
- `figure_question`: what question this figure answers
- `author_claim`: what the authors use this figure to argue
- `viewer_takeaway`: what the viewer should remember
- `caption_evidence`: caption lines or summarized caption facts used
- `results_evidence`: nearby Results text used
- `panels`: important panel-by-panel explanations
- `caveat`: what the figure does not prove

For each important panel, capture:

- panel label
- what is plotted
- axes or measured variables
- groups, colors, conditions, or alignment event
- what comparison is being made
- what visible pattern appears
- how that pattern supports or limits the figure claim
- whether the video should show a full figure, crop, zoom, highlight box, or animation

## Narration Formula

Use this pattern:

```text
This panel plots [measured variable] against [time/condition/model/etc.].
[Color/group/line] means [condition].
The key comparison is [A versus B].
The pattern to notice is [visible result].
That supports the claim because [reason].
It does not by itself show [caveat], so the next panel/figure checks [next step].
```

## Failure Modes

Reject narration that:

- says only "they found X" without explaining the plot
- ignores axes, variables, groups, colors, or alignment
- reads the caption aloud instead of translating it
- describes author inference as direct observation
- skips the task/setup needed to understand the plot
- uses a full-page figure when the viewer needs a panel zoom

## Scene Rule

For each main figure in scope, create at least:

- one setup scene explaining why the figure exists
- one or more panel scenes explaining actual plotted evidence
- one claim scene explaining what the figure contributes to the paper
